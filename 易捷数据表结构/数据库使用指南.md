## 一、快速开始

## 二、常用查询示例

### 2.1 获取部门列表（从易捷集团）
```sql
SELECT 
    dept_cde AS 部门代码,
    dept_nme AS 部门名称,
    is_active AS 是否有效,
    quence AS 排序
FROM ferp.pb_dept
WHERE is_active = 'Y'
ORDER BY quence;
```

### 2.2 获取业务员列表（从易捷集团）
```sql
SELECT 
    d.dept_cde AS 部门代码,
    d.dept_nme AS 部门名称,
    m.user_cde AS 业务员代码,
    m.user_nme AS 业务员姓名,
    m.mobile AS 手机,
    m.is_active AS 是否有效
FROM ferp.pb_dept d
LEFT JOIN ferp.pb_dept_member m ON d.dept_cde = m.dept_cde
WHERE d.is_active = 'Y'
  AND m.is_active = 'Y'
ORDER BY d.quence, m.quence;
```

### 2.3 查询客户基本信息
```sql
SELECT 
    clntcde AS 客户编码,
    clntnme AS 客户名称,
    smpnme AS 客户简称,
    clntyp AS 客户类型,
    agntcde AS 默认销售员,
    tel AS 电话,
    address AS 地址
FROM ferp.pb_clnt
WHERE isactive = 'Y'
  AND clntnme LIKE '%{关键词}%'
ORDER BY clntcde;
```

### 2.4 查询订单信息
```sql
SELECT 
    serial AS 生产单号,
    ctpono AS 客户订单号,
    ctline AS 订单行号,
    refcde AS 客户料号,
    refnme AS 客户品名,
    ctlnum AS 数量,
    ordtyp AS 订单类型,
    created AS 创建时间
FROM ferp.ord_ct
WHERE isactive = 'Y'
  AND created >= DATE '2026-01-01'
ORDER BY created DESC;
```

### 2.5 查询纸板报价
```sql
SELECT 
    matcde AS 纸质,
    invcde AS 客户材质,
    crrcde AS 楞别,
    layers AS 层数,
    prices AS 报价,
    fixfee AS 开机费,
    minamt AS 保底,
    remark AS 备注
FROM ferp.mk_prctyp01
WHERE isactive = 'Y'
  AND clientid = '{客户 ID}'
ORDER BY matcde, crrcde;
```

### 2.6 查询水印报价
```sql
SELECT 
    prcnme AS 工序名称,
    mactyp AS 机型,
    prtacr AS 印刷面积,
    colsum AS 颜色数,
    tptype AS 模板类型,
    caltyp AS 计算方式,
    prices AS 价格,
    fixfee AS 开机费,
    minamt AS 保底
FROM ferp.mk_prctyp02
WHERE isactive = 'Y'
  AND prcmod IN ('FIX156', 'LP', 'PI')
ORDER BY prcnme, colsum;
```

## 三、表结构说明

### 3.1 通用字段说明

以下字段在所有表中通用：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` | Integer | 主键 ID |
| `created` | Date | 创建时间 |
| `createdby` | VARCHAR(11) | 创建人 |
| `updated` | Date | 更新时间 |
| `updatedby` | VARCHAR(11) | 更新人 |
| `orgcde` | VARCHAR(5) | 组织代码 |
| `clientid` | VARCHAR(5) | 客户 ID |
| `isactive` | VARCHAR(1) | 是否有效 (Y/N) |

### 3.2 常用代码说明

#### 订单类型 (`ordtyp`)
- `G` = 正常订单
- `D` = 打样订单
- `S` = 补单
- `R` = 返修
- `B` = 备库

#### 客户类型 (`clntyp`)
- 参考 `sys_dict` 数据字典表

#### 计价方式 (`prcmod`)
- `FIX156` = 固定价格
- `LP` = 加工费
- `PI` = 计件
- `WX` = 外协
- `CO` = 成本计算

#### 计算方式 (`caltyp`)
- `PCS` = 按个计算
- `ACR` = 按面积计算

## 四、最佳实践

### 4.1 查询优化

1. **始终使用 `isactive = 'Y'` 过滤**
   ```sql
   WHERE isactive = 'Y'  -- 只查询有效数据
   ```

2. **使用索引字段**
   - `clientid` - 客户 ID
   - `clntcde` - 客户编码
   - `serial` - 生产单号
   - `created` - 创建时间

3. **避免全表扫描**
   ```sql
   -- ✅ 好的查询
   WHERE clientid = 'C0001'
   
   -- ❌ 避免的查询
   WHERE refnme LIKE '%关键词%'  -- 前缀匹配无法使用索引
   ```

### 4.2 数据安全

1. **使用只读账号**
   - 生产环境使用 `read` 账号
   - 避免使用管理员账号

2. **不要修改数据**
   - 仅执行 `SELECT` 查询
   - 禁止 `INSERT/UPDATE/DELETE`

3. **敏感数据脱敏**
   - 密码字段 (`passwd`) 不要查询
   - 客户财务信息谨慎处理

### 4.3 错误处理

```csharp
try
{
    // 执行查询
    var result = await QueryAsync(sql);
}
catch (OracleException ex)
{
    // 记录错误日志
    Console.WriteLine($"数据库错误：{ex.Message}");
    
    // 重试逻辑（如果是临时错误）
    if (ex.Number == 12170 || ex.Number == 12560)  // 连接超时
    {
        // 重试...
    }
}
```

## 五、常见问题

### Q1: 为什么部门和业务员数据不准确？
**A**: 请确保从易捷集团数据库 (`fgrp`) 查询：
- 主机：`36.138.130.91`
- 表名：`ferp.pb_dept` 和 `ferp.pb_dept_member`

### Q2: 如何获取最新的报价？
**A**: 报价可能有多个版本，按 `updated` 降序取第一条：
```sql
SELECT * FROM (
    SELECT * FROM ferp.mk_prctyp01
    WHERE clientid = 'C0001'
      AND matcde = 'A123'
    ORDER BY updated DESC
)
WHERE ROWNUM = 1;
```

### Q3: 如何关联客户和订单？
**A**: 使用 `clientid` 或 `clntcde` 关联：
```sql
SELECT c.clntnme, o.serial, o.ctlnum
FROM ferp.pb_clnt c
JOIN ferp.ord_ct o ON c.clientid = o.clientid
WHERE c.isactive = 'Y'
  AND o.isactive = 'Y';
```

### Q4: 数据库连接超时怎么办？
**A**: 
1. 检查网络连接
2. 增加连接超时时间
3. 使用连接池
4. 检查防火墙设置
