## 修复内容总结

### 1. 模块_通用函数.cs - pb_dept_member 查询修复 ✅

**问题**：
- 使用了错误的字段名 `is_active`，应该是 `isactive`
- 没有连接 pb_dept 表获取部门名称
- TEMCDE 字段可能未正确填充

**修复方案**：
```csharp
// 修复前
SELECT user_cde as EMPCDE, dept_cde as TEMCDE, dept_cde as TEMCDE2, user_nme as EMPNME, '销售' as TEMNME
FROM pb_dept_member
WHERE is_active = 'Y'
ORDER BY user_nme

// 修复后
SELECT m.user_cde as EMPCDE, m.dept_cde as TEMCDE, m.dept_cde as TEMCDE2, 
       m.user_nme as EMPNME, d.dept_nme as TEMNME
FROM pb_dept_member m
LEFT JOIN pb_dept d ON m.dept_cde = d.dept_cde
WHERE m.isactive = 'Y'
ORDER BY d.dept_nme, m.user_nme
```

同时在数据合并逻辑中补充了 TEMCDE 字段的填充：

```csharp
if (temp.TEMCDE == null || temp.TEMCDE == "") {
    temp.TEMCDE = t.TEMCDE;
}
```

### 2. 窗体_利润统计.cs - SQL 修复 ✅

**问题**：
- 查询中使用了 `hr_base` 表但没有 JOIN
- 部门筛选条件使用 `d.dptcde` 但 pb_dept 表的字段可能是 `dept_cde`

**修复方案**：
```csharp
// 修复前
FROM ord_bas b
LEFT JOIN pb_clnt c ON b.clntcde = c.clntcde
LEFT JOIN ord_ct t ON b.serial = t.serial
LEFT JOIN hr_base h ON t.agntcde = h.mobile
LEFT JOIN pb_dept d ON h.dptcde = d.dptcde
...
AND d.dptcde IN (...)

// 修复后
FROM ord_bas b
LEFT JOIN pb_clnt c ON b.clntcde = c.clntcde
LEFT JOIN ord_ct t ON b.serial = t.serial
LEFT JOIN pb_dept_member e ON t.agntcde = e.mobile
LEFT JOIN pb_dept d ON e.dept_cde = d.dept_cde
...
AND d.dept_cde IN (...)
```
