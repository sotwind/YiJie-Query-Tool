### pb_dept_member 表结构

根据文档 `数据库使用指南.md` 中的 SQL 示例：

```sql
SELECT 
    d.dept_cde AS 部门代码,
    d.dept_nme AS 部门名称,
    m.user_cde AS 业务员代码,
    m.user_nme AS 业务员姓名,
    m.mobile AS 手机,
    m.is_active AS 是否有效
FROM ferp.pb_dept d
LEFT JOIN ferp.pb_dept_member m ON d.dept_cde = m.dept_cde
```

| 字段名 | 类型 | 说明 |
|--------|------|------|
| dept_cde | VARCHAR | 部门编码 |
| user_cde | VARCHAR | 业务员编码 |
| user_nme | VARCHAR | 业务员姓名 |
| mobile | VARCHAR | 手机号码 |
| is_active / isactive | VARCHAR | 是否有效 |

## 代码问题分析

### 问题1: pb_dept_member 字段名错误

**当前代码**：
```csharp
SELECT user_cde as EMPCDE, dept_cde as TEMCDE, dept_cde as TEMCDE2, user_nme as EMPNME, '销售' as TEMNME
FROM pb_dept_member
WHERE is_active = 'Y'
ORDER BY user_nme
```

**问题**：
- `user_nme` 字段可能不存在，应该是 `user_nme` 或 `empnme`
- `is_active` 应该是 `isactive` (without underscore)

**修复方案**：
```csharp
SELECT user_cde as EMPCDE, dept_cde as TEMCDE, dept_cde as TEMCDE2, user_nme as EMPNME, d.dept_nme as TEMNME
FROM pb_dept_member m
LEFT JOIN pb_dept d ON m.dept_cde = d.dept_cde
WHERE m.isactive = 'Y'
ORDER BY d.dept_nme, m.user_nme
```

### 问题2: 利润统计部门筛选无效

**当前代码**：
```csharp
string 部门编码 = rowview["TEMCDE"].ToString();
if (部门编码 != "" && 部门编码 != null)
{
    if (tmpstr != "") tmpstr += ",";
    tmpstr += "'" + 部门编码 + "'";
}
sql += @" AND d.dptcde IN (" + tmpstr + ")";
```

**问题**：
- `TEMCDE` 来自 `pb_dept_member` 表的 `dept_cde`
- `pb_dept` 表的部门编码字段可能是 `dptcde` 或 `dept_cde`
- 需要确认两个表的关联字段是否一致

**修复方案**：
需要先修复 pb_dept_member 查询，确保 TEMCDE 字段正确填充。
