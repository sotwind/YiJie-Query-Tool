# 利润统计查询无数据问题分析

## 问题描述

用户反馈点击查询按钮后，一条数据也没有显示。

## 代码分析

### 查询流程

1. 点击"查询"按钮触发 `按钮_查询_Click` 事件
2. 调用 `BuildQueryString()` 构建 SQL 查询
3. 遍历所有数据库连接（只处理"新系统"类型）
4. 执行查询并将结果添加到 `profitDataList`
5. 调用 `显示结果 (profitDataList)` 显示数据

### 可能的问题原因

#### 1. **部门筛选条件过于严格** ⭐⭐⭐ (最可能)

从截图可以看到：

* **销售 1 部** 被勾选（蓝色高亮，有勾选标记）

* 但是**业务员列表中没有显示对应的业务员**

查看代码逻辑：

```csharp
// 在 BuildQueryString() 中
if (列表_部门.CheckedItems.Count > 0)
{
    // 构建部门编码 IN 条件
    sql += @" AND d.dptcde IN (" + tmpstr + @")";
}
```

**问题**：当勾选"销售 1 部"时，SQL 会添加 `d.dptcde IN (...)` 条件，但：

* 部门编码可能为空或不匹配

* 业务员和部门的关联可能有问题

* 导致 JOIN 后没有数据

#### 2. **部门编码为空的问题**

在 `BuildQueryString()` 第 114 行：

```csharp
string 部门编码 = rowview["TEMCDE"].ToString();
if (部门编码 != "" && 部门编码 != null)
```

查看 `易捷部门表 ()` 的数据来源：

* 部门数据来自 `get 业务员表 ()` 方法

* 该方法先从易捷集团数据库查询，再补充各业务数据库

* 在合并数据时，可能存在 `TEMCDE` 为空的情况

**关键代码**（第 128-137 行）：

```csharp
if (dict.TryGetValue(t.TEMNME + "-" + t.EMPNME, out temp)) {
    if (temp.EMPCDE2 == null || temp.EMPCDE2 == "") {
        temp.EMPCDE2 = t.EMPCDE2;
    }
    if (temp.EMPCDE == null || temp.EMPCDE == "") {
        temp.EMPCDE = t.EMPCDE;
    }
    if (temp.TEMCDE2 == null || temp.TEMCDE2 == "") {
        temp.TEMCDE2 = t.TEMCDE2;  // 只填充 TEMCDE2
    }
}
```

**发现问题**：`TEMCDE`（部门编码）可能没有被正确填充！

#### 3. **日期范围问题**

* 默认日期：2026 年 3 月 4 日 到 2026 年 3 月 5 日

* 这个日期是未来时间，数据库中可能没有数据

#### 4. **数据库连接问题**

代码只查询"新系统"类型的数据库：

```csharp
if (db.ServerType != "新系统")
    continue;
```

当前配置的新系统数据库：

* 新厂新系统 (36.134.7.141)

* 老厂新系统 (36.138.132.30)

* 温森新系统 (db.05.forestpacking.com)

如果这些数据库连接失败或没有数据，就不会有结果。

#### 5. **SQL JOIN 条件问题**

```sql
FROM ord_bas b
LEFT JOIN pb_clnt c ON b.clntcde = c.clntcde
LEFT JOIN ord_ct t ON b.serial = t.serial
LEFT JOIN hr_base h ON t.agntcde = h.mobile
LEFT JOIN pb_dept d ON h.dptcde = d.dptcde
```

**潜在问题**：

* `ord_ct` 表的 `agntcde` 字段与 `hr_base` 的 `mobile` 关联

* 如果 `agntcde` 为空或不匹配，会导致 LEFT JOIN 后部门信息为空

* 当添加 `d.dptcde IN (...)` 条件时，实际上把 LEFT JOIN 变成了 INNER JOIN

## 根本原因推断

**最可能的原因**：

1. **部门编码 TEMCDE 为空**：在 `get 业务员表 ()` 方法中，从易捷集团数据库查询的数据没有 `TEMCDE` 字段，只填充了 `TEMCDE2`
2. **部门筛选条件失效**：当勾选部门时，实际传入 SQL 的部门编码可能是空字符串
3. **日期是未来时间**：2026 年的数据在数据库中不存在

## 修复方案

### 方案 1：修复部门编码获取逻辑 ✅ 已实施

**问题根源**：
- 从 `PB_DEPT_MEMBER` 表查询时，SQL 只返回了 `DPTCDE as TEMCDE2`
- `empTemp` 类的 `TEMCDE` 属性没有被赋值，始终为 null
- 导致部门表 (`dptTemp`) 的 `TEMCDE` 字段也为 null
- 在查询筛选时，由于 `TEMCDE` 为空，无法构建有效的 `IN` 条件

**修复内容**：
1. 修改 SQL 查询，添加 `PB_DEPT` 表的 JOIN 获取部门名称和编码：
```sql
SELECT m.EMPCDE, m.DPTCDE as TEMCDE, m.DPTCDE as TEMCDE2, m.EMPNME, d.DPTNME as TEMNME
FROM PB_DEPT_MEMBER m
LEFT JOIN PB_DEPT d ON m.DPTCDE = d.DPTCDE
WHERE m.ISACTIVE = 'Y'
ORDER BY d.DPTNME, m.EMPNME
```

2. 在数据合并逻辑中，添加 `TEMCDE` 字段的填充：
```csharp
if (temp.TEMCDE == null || temp.TEMCDE == "") {
    temp.TEMCDE = t.TEMCDE;
}
```
