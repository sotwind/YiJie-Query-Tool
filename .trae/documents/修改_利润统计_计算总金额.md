# 修改计划：利润统计计算总金额

## 用户需求

1. 当前显示的是单价（报价金额、卖价金额）
2. 需要改为显示总金额（单价 × 数量）
3. 差额改为"毛利"更准确

## 需要确认的字段

需要查询 ord_bas/ord_ct/ORD_CT 表中是否有数量字段：
- 新系统：ord_bas 或 ord_ct 中是否有数量字段？
- 旧系统：ORD_CT 中是否有数量字段？

## 修改方案

### 方案1：SQL 中直接计算
如果表中有数量字段（如 qty、quantity、num 等）：
```sql
-- 新系统
NVL(b.quoprc, 0) * NVL(b.qty, 1) as 报价总金额,
NVL(b.prices, 0) * NVL(b.qty, 1) as 卖价总金额,
NVL(b.prices, 0) * NVL(b.qty, 1) - NVL(b.quoprc, 0) * NVL(b.qty, 1) as 毛利,

-- 旧系统
NVL(ct.AGNTPRC, 0) * NVL(ct.QTY, 1) as 报价总金额,
NVL(ct.PRICES, 0) * NVL(ct.QTY, 1) as 卖价总金额,
NVL(ct.PRICES, 0) * NVL(ct.QTY, 1) - NVL(ct.AGNTPRC, 0) * NVL(ct.QTY, 1) as 毛利,
```

### 方案2：先查询确认字段名
需要查询数据库确认数量字段的确切名称。

## 实施步骤

1. 查询新系统和旧系统的数量字段名
2. 修改 SQL 计算总金额
3. 修改列标题："差额" → "毛利"
4. 更新汇总计算
