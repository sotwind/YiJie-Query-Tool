# 诊断 ORA-01427 错误计划

## 问题描述

用户在利润统计功能中，选择日期范围（2月1日到3月5日）和销售1部后点击查询，新厂系统出现错误：
- **错误代码**：ORA-01427
- **错误信息**：单行子查询返回多个行

## 诊断目标

1. 找出具体是哪个（或哪些）客户编码导致子查询返回多行
2. 查看这些重复记录的具体值是什么
3. 确定修复方案：修改子查询只取一条记录

## 诊断步骤

### 步骤1：查询 V_ORD 视图定义

首先查看 `V_ORD` 视图的完整定义，确认哪些字段使用了标量子查询。

```sql
SELECT view_name, text
FROM user_views
WHERE view_name = 'V_ORD'
```

### 步骤2：找出有重复的客户编码

查询 `CLNT` 表中哪些 `CLNTCDE` 有重复记录：

```sql
SELECT CLNTCDE, COUNT(*) as cnt
FROM CLNT
GROUP BY CLNTCDE
HAVING COUNT(*) > 1
ORDER BY cnt DESC
FETCH FIRST 20 ROWS ONLY
```

### 步骤3：查看重复客户的具体数据

针对有重复的客户编码，查看具体重复的记录内容：

```sql
SELECT *
FROM CLNT
WHERE CLNTCDE IN (
    SELECT CLNTCDE
    FROM CLNT
    GROUP BY CLNTCDE
    HAVING COUNT(*) > 1
)
ORDER BY CLNTCDE
```

### 步骤4：结合查询条件缩小范围

根据用户的查询条件（日期范围、销售1部业务员），找出实际触发错误的客户：

```sql
-- 先找出销售1部涉及的业务员编码
SELECT DISTINCT o.AGNTCDE
FROM ORD o
WHERE o.PTDATE >= TO_DATE('2026-02-01', 'yyyy-MM-dd')
  AND o.PTDATE < TO_DATE('2026-03-06', 'yyyy-MM-dd')
  AND o.STATUS = 'Y'
  AND EXISTS (
      SELECT 1 FROM PB_EMPS e 
      WHERE e.EMPCDE = o.AGNTCDE 
      AND e.TEMCDE = '销售1部对应的部门编码'
  )

-- 然后检查这些业务员的订单中，哪些客户有重复
SELECT o.CLNTCDE, c.CLNTNME, COUNT(DISTINCT c.ROWID) as clnt_count
FROM ORD o
JOIN CLNT c ON o.CLNTCDE = c.CLNTCDE
WHERE o.PTDATE >= TO_DATE('2026-02-01', 'yyyy-MM-dd')
  AND o.PTDATE < TO_DATE('2026-03-06', 'yyyy-MM-dd')
  AND o.STATUS = 'Y'
  AND o.AGNTCDE IN (/* 销售1部的业务员编码 */)
GROUP BY o.CLNTCDE, c.CLNTNME
HAVING COUNT(DISTINCT c.ROWID) > 1
```

## 修复方案（诊断后实施）

确认问题客户后，修改 `V_ORD` 视图中的子查询，使用 `ROWNUM = 1` 或聚合函数只取一条记录：

```sql
-- 原查询（有问题）
(SELECT CLNTNME FROM CLNT WHERE CLNTCDE = o.CLNTCDE) AS CLNTNME

-- 修复方案1：使用 ROWNUM
(SELECT CLNTNME FROM CLNT WHERE CLNTCDE = o.CLNTCDE AND ROWNUM = 1) AS CLNTNME

-- 修复方案2：使用 MAX/MIN
(SELECT MAX(CLNTNME) FROM CLNT WHERE CLNTCDE = o.CLNTCDE) AS CLNTNME
```

## 实施计划

1. **诊断阶段**：执行上述 SQL 查询，找出问题客户
2. **分析阶段**：查看重复记录的具体值，分析产生重复的原因
3. **修复阶段**：修改应用程序中的 SQL，在子查询中添加 `ROWNUM = 1` 限制
