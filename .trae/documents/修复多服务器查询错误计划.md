# 修复多服务器查询错误计划

## 问题分析

### 错误1：销售员图 - ORA-00904: "OBJTYP": 标识符无效

* **原因**：`ord_ct` 表中没有 `objtyp` 字段

* **影响**：所有4个服务器都报错

### 错误2：利润统计 - ORA-00942: 表或视图不存在

* **原因**：`ord_ct` 表在某些服务器上不存在（如临海）

* **影响**：3个服务器报错，临海没弹（可能是表不存在）

## 根本原因

不同服务器的数据库结构不同：

1. **新系统**（新厂）：使用 `ord_ct` 表，但没有 `objtyp` 字段
2. **旧系统**（老厂、临海等）：使用 `ord` 表

## 修复方案

### 方案1：根据服务器类型使用不同的表名（推荐）

#### 销售员图修复

```csharp
// 根据数据库连接判断使用哪个表
if (db.ServerType == "新系统") {
    // 新系统使用 ord_ct，但需要确认字段名
    sql = "select ... from ord_ct where ...";
} else {
    // 旧系统使用 v_ord 或 ord
    sql = "select objtyp, agntcde, ... from v_ord where ...";
}
```

#### 利润统计修复

```csharp
// 同样需要区分新旧系统
if (db.ServerType == "新系统") {
    sql = "SELECT ... FROM ord_ct ...";
} else {
    sql = "SELECT ... FROM ord ...";
}
```

### 方案2：检查并统一数据库结构

* 在所有服务器上创建一致的表结构

* 但这需要DBA配合，周期较长

## 实施步骤

### 步骤1：查看 DatabaseInfos 配置

* 确认如何区分新旧系统

* 查看 ServerType 字段

### 步骤2：查看 ord\_ct 表结构

* 确认新系统 ord\_ct 表的字段名

* 找到对应 objtyp 的字段

### 步骤3：修改销售员图查询

* 根据 ServerType 使用不同的SQL

* 新系统：使用 ord\_ct，调整字段名

* 旧系统：继续使用 v\_ord

### 步骤4：修改利润统计查询

* 同样根据 ServerType 区分

* 处理表不存在的问题

### 步骤5：测试验证

* 在所有4个服务器上测试

* 确认错误已解决
