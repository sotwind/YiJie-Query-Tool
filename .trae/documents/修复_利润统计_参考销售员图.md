# 修复计划：利润统计参考销售员图实现

## 用户建议

不需要去查询 pb\_dept 表的正确字段名，应该参考销售员图窗体的实现方式：

* 销售员图已经正确实现了业务员数据的获取

* 利润统计应该使用相同的逻辑

## 销售员图的实现方式

在销售员图中：

1. 使用 `模块_通用函数.易捷业务员表()` 获取所有业务员数据
2. 业务员数据已经包含了部门信息（TEMNME）
3. 不需要在 SQL 中关联部门表

## 利润统计的问题

当前利润统计的 SQL 复杂化了：

```sql
LEFT JOIN pb_dept_member e ON t.agntcde = e.empcde
LEFT JOIN pb_dept d ON e.dept_cde = d.dept_cde
```

## 修复方案

简化 SQL，不在 SQL 中关联业务员和部门表：

1. 只查询订单数据（ord\_bas/ord）
2. 在 C# 代码中通过 `agntcde` 去查找对应的业务员姓名和部门
3. 使用 `模块_通用函数.易捷业务员表()` 获取的业务员字典进行查找

## 具体修改

修改 `BuildQueryString` 方法：

1. 移除业务员表和部门表的 LEFT JOIN
2. 只保留订单相关表的查询
3. 在 `显示结果` 方法中，根据 `agntcde` 去业务员表中查找对应的姓名和部门
