# 修复计划：利润统计部门表字段错误

## 问题分析

从截图可以看到错误信息：
```
老厂新系统 连接出错了：ORA-00904: "D"."DEPT_CDE": 标识符无效
```

SQL 中使用了错误的字段名：
- `d.dept_cde` - 错误，pb_dept 表中没有这个字段

## 原因

在 [窗体_利润统计.cs](file:///h:/TraeDev/易捷业务数据查询/易捷查询CSharp/易捷查询CSharp/窗体_利润统计.cs) 中，对于老厂新系统：
- 部门表使用 `pb_dept`
- 但关联条件使用了 `e.dept_cde = d.dept_cde`
- 可能 pb_dept 表的字段名不是 dept_cde

## 需要确认的问题

1. 老厂新系统的 `pb_dept` 表实际字段名是什么？
2. `pb_dept_member` 表中的部门编码字段名是什么？

## 修复方案

需要查询老厂新系统的 `pb_dept` 表结构，确认正确的字段名。

可能的字段名：
- `dptcde` 而不是 `dept_cde`
- `dptnme` 而不是 `dept_nme`

## 实施步骤

1. 查询老厂新系统 `pb_dept` 表结构
2. 修改 SQL 中的字段名
