# 分析重复客户数据计划

## 问题现状

修改后的 SQL 出现新错误：**ORA-04044: 此处不允许过程, 函数, 程序包或类型**

原因：Oracle 不允许在 SELECT 子句中使用带有 `ROWNUM` 的子查询作为列表达式。

## 目标

1. 找出具体哪些 `CLNTCDE` 返回了多行客户名称
2. 显示这些重复数据的具体内容，供用户分析
3. 修复 SQL 查询，使用其他方式避免 ORA-01427 错误

## 诊断步骤

### 步骤1：查询 CLNT 表中重复的客户编码及数据

```sql
-- 找出重复的客户编码
SELECT CLNTCDE, COUNT(*) as cnt
FROM CLNT
GROUP BY CLNTCDE
HAVING COUNT(*) > 1
ORDER BY cnt DESC;

-- 查看重复客户的详细数据
SELECT *
FROM CLNT
WHERE CLNTCDE IN (
    SELECT CLNTCDE
    FROM CLNT
    GROUP BY CLNTCDE
    HAVING COUNT(*) > 1
)
ORDER BY CLNTCDE;
```

### 步骤2：结合查询条件找出实际触发错误的客户

根据用户的查询条件（日期范围、销售1部业务员），找出实际触发错误的客户：

```sql
-- 先找出销售1部涉及的业务员编码
SELECT DISTINCT EMPCDE, EMPNME
FROM PB_EMPS
WHERE TEMNME LIKE '%销售1部%';

-- 然后检查这些业务员的订单中，哪些客户有重复
SELECT o.CLNTCDE, 
       (SELECT CLNTNME FROM CLNT WHERE CLNTCDE = o.CLNTCDE) as CLNTNME,
       COUNT(DISTINCT c.ROWID) as clnt_count
FROM ORD o
JOIN CLNT c ON o.CLNTCDE = c.CLNTCDE
WHERE o.PTDATE >= TO_DATE('2026-02-01', 'yyyy-MM-dd')
  AND o.PTDATE < TO_DATE('2026-03-06', 'yyyy-MM-dd')
  AND o.STATUS = 'Y'
  AND o.AGNTCDE IN (/* 销售1部的业务员编码 */)
GROUP BY o.CLNTCDE
HAVING COUNT(DISTINCT c.ROWID) > 1;
```

## 修复方案

### 方案1：使用 LEFT JOIN + 聚合（推荐）

不使用子查询，改用 LEFT JOIN 配合聚合函数：

```sql
SELECT 
    TO_CHAR(o.PTDATE, 'yyyy-MM-dd') as 日期,
    o.SERIAL as 单号,
    MAX(c.CLNTNME) as 客户,  -- 使用聚合函数
    MAX(p.PRDNME) as 产品,   -- 使用聚合函数
    o.AGNTCDE as 业务员编码,
    NVL(b.QUOPRC, 0) * NVL(o.ACCNUM, 0) as 报价总金额,
    NVL(o.PRICES, 0) * NVL(o.ACCNUM, 0) as 卖价总金额,
    NVL(o.PRICES, 0) * NVL(o.ACCNUM, 0) - NVL(b.QUOPRC, 0) * NVL(o.ACCNUM, 0) as 毛利,
    CASE 
        WHEN NVL(b.QUOPRC, 0) * NVL(o.ACCNUM, 0) = 0 THEN NULL
        ELSE ROUND((NVL(o.PRICES, 0) * NVL(o.ACCNUM, 0) - NVL(b.QUOPRC, 0) * NVL(o.ACCNUM, 0)) / (NVL(b.QUOPRC, 0) * NVL(o.ACCNUM, 0)) * 100, 2)
    END as 利率
FROM ORD o
LEFT JOIN (
    -- 使用 GROUP BY 去重，每个 CLNTCDE 只取一条
    SELECT CLNTCDE, MAX(CLNTNME) as CLNTNME
    FROM CLNT
    GROUP BY CLNTCDE
) c ON o.CLNTCDE = c.CLNTCDE
LEFT JOIN (
    -- 使用 GROUP BY 去重，每个 PRDCDE 只取一条
    SELECT PRDCDE, MAX(PRDNME) as PRDNME
    FROM PRD
    GROUP BY PRDCDE
) p ON o.PRDCDE = p.PRDCDE
LEFT JOIN ORD_BAS b ON o.SERIAL = b.SERIAL AND o.CLIENTID = b.CLIENTID AND o.ORGCDE = b.ORGCDE
WHERE o.STATUS = 'Y'
  AND o.PTDATE >= to_date('2026-02-01', 'yyyy-MM-dd')
  AND o.PTDATE < to_date('2026-03-06', 'yyyy-MM-dd')
GROUP BY o.PTDATE, o.SERIAL, o.AGNTCDE, o.ACCNUM, o.PRICES, b.QUOPRC
ORDER BY o.PTDATE DESC
```
