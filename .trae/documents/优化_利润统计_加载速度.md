# 优化计划：利润统计加载速度优化

## 问题分析

从截图看，数据已经能正常加载，但用户反馈加载很慢。

## 性能瓶颈分析

在 [窗体_利润统计.cs](file:///h:/TraeDev/易捷业务数据查询/易捷查询CSharp/易捷查询CSharp/窗体_利润统计.cs) 的 `利润统计数据` 类中：

```csharp
public string 业务员 
{ 
    get 
    { 
        // 每次访问都查询业务员表
        var 业务员表 = 模块_通用函数.易捷业务员表();
        var rows = 业务员表.Select($"EMPCDE='{业务员编码}' OR EMPCDE2='{业务员编码}'");
        ...
    } 
}

public string 部门 
{ 
    get 
    { 
        // 每次访问都查询业务员表
        var 业务员表 = 模块_通用函数.易捷业务员表();
        var rows = 业务员表.Select($"EMPCDE='{业务员编码}' OR EMPCDE2='{业务员编码}'");
        ...
    } 
}
```

**问题**：
1. 每次访问 `业务员` 和 `部门` 属性时都会调用 `模块_通用函数.易捷业务员表()`
2. 每次都要执行 `Select` 查询
3. 在 `显示结果` 方法中，每行数据都要访问这两个属性多次

## 优化方案

### 方案1：使用缓存（推荐）
在 `显示结果` 方法中：
1. 预先获取业务员表
2. 创建业务员编码到姓名/部门的字典缓存
3. 在显示时直接从缓存查找

### 方案2：延迟加载 + 缓存
在 `利润统计数据` 类中：
1. 添加私有字段缓存结果
2. 只在第一次访问时查询

### 方案3：批量处理
在查询数据后：
1. 一次性获取所有需要的业务员信息
2. 使用字典进行快速查找

## 实施步骤

1. 修改 `显示结果` 方法，添加业务员信息缓存
2. 修改 `利润统计数据` 类，使用缓存的字典
3. 或者直接在 SQL 查询时优化（如果可能）

## 预期效果
- 从 O(n²) 降到 O(n) 的时间复杂度
- 大幅提升数据加载速度
