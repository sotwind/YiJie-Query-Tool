# 修复 ORA-01427 错误计划

## 问题描述

用户在利润统计功能中，选择日期范围（2月1日到3月5日）和部门（销售1部）后点击查询，出现错误：
- **错误代码**：ORA-01427
- **错误信息**：单行子查询返回多个行

## 问题分析

### 根本原因
`V_ORD` 视图内部定义包含标量子查询（Scalar Subquery），当子查询条件匹配到多条记录时，就会触发 ORA-01427 错误。

例如，视图定义可能类似：
```sql
CREATE VIEW V_ORD AS
SELECT 
    o.SERIAL,
    o.PTDATE,
    (SELECT CLNTNME FROM CLNT WHERE CLNTCDE = o.CLNTCDE) AS CLNTNME,  -- 这里可能返回多行
    ...
FROM ORD o
```

当 `CLNT` 表中存在多条相同 `CLNTCDE` 的记录时，上述子查询就会返回多行，导致错误。

### 当前SQL分析
当前利润统计查询：
```sql
SELECT 
    TO_CHAR(v.PTDATE, 'yyyy-MM-dd') as 日期,
    v.SERIAL as 单号,
    v.CLNTNME as 客户,
    ...
FROM V_ORD v
LEFT JOIN ORD_BAS b ON v.SERIAL = b.SERIAL AND v.CLIENTID = b.CLIENTID AND v.ORGCDE = b.ORGCDE
WHERE v.ISACTIVE = 'Y'
  AND v.PTDATE >= to_date('2026-02-01', 'yyyy-MM-dd')
  AND v.PTDATE < to_date('2026-03-06', 'yyyy-MM-dd')
  AND v.AGNTCDE IN (...)
ORDER BY v.PTDATE DESC
```

## 修复方案

### 方案：绕过 V_ORD 视图，直接查询基础表

由于无法修改数据库中的视图定义，我们在应用程序中绕过 `V_ORD` 视图，直接查询基础表 `ORD` 和相关表，避免使用包含问题子查询的视图。

#### 新系统 SQL 修改

将原来的：
```sql
FROM V_ORD v
LEFT JOIN ORD_BAS b ON v.SERIAL = b.SERIAL AND v.CLIENTID = b.CLIENTID AND v.ORGCDE = b.ORGCDE
WHERE v.ISACTIVE = 'Y'
```

改为直接查询基础表：
```sql
FROM ORD o
LEFT JOIN (
    -- 使用聚合确保每个订单只返回一行报价数据
    SELECT SERIAL, CLIENTID, ORGCDE, MAX(QUOPRC) as QUOPRC
    FROM ORD_BAS
    GROUP BY SERIAL, CLIENTID, ORGCDE
) b ON o.SERIAL = b.SERIAL AND o.CLIENTID = b.CLIENTID AND o.ORGCDE = b.ORGCDE
LEFT JOIN CLNT c ON o.CLNTCDE = c.CLNTCDE
LEFT JOIN PB_EMPS e ON o.AGNTCDE = e.EMPCDE
WHERE o.STATUS = 'Y'
```

#### 旧系统 SQL 修改

类似地修改旧系统的查询，将 `ORD_CT` 表的 JOIN 也使用聚合子查询。

## 实施步骤

1. **修改 `窗体_利润统计.cs` 中的 `BuildQueryString` 方法**
   - 新系统分支：重写 SQL，直接查询 `ORD` 表而非 `V_ORD` 视图
   - 旧系统分支：同样重写 SQL，直接查询 `ORD` 表

2. **字段映射调整**
   - `V_ORD` 视图中的字段名需要映射到基础表的字段名
   - 主要映射关系：
     - `V_ORD.ISACTIVE` → `ORD.STATUS`
     - `V_ORD.PTDATE` → `ORD.PTDATE`
     - `V_ORD.SERIAL` → `ORD.SERIAL`
     - `V_ORD.CLNTNME` → `CLNT.CLNTNME` (通过 JOIN 获取)
     - `V_ORD.PRDNME` → 需要确定来源表
     - `V_ORD.AGNTCDE` → `ORD.AGNTCDE`
     - `V_ORD.ACCNUM` → `ORD.ACCNUM`
     - `V_ORD.PRICES` → `ORD.PRICES`

3. **测试验证**
   - 使用相同的查询条件（2月1日-3月5日，销售1部）进行测试
   - 验证查询结果是否正确
   - 验证性能是否可接受

## 风险与注意事项

1. **字段名差异**：根据 `oracle.md` 规则，不同系统的数据库字段名可能存在差异，需要逐一验证。
2. **性能影响**：直接查询基础表可能涉及更多 JOIN 操作，需要关注查询性能。
3. **数据一致性**：确保新的查询方式与原来的视图查询返回一致的结果。
