# 利润统计 - 部门筛选业务员功能计划

## 需求描述
在"利润统计"窗口中，当用户勾选某个销售部门时，业务员列表只显示该部门的业务员，而不是显示所有业务员。

## 当前问题
- 业务员列表显示所有业务员，人员太多，查找困难
- 部门和业务员列表之间没有联动关系

## 解决方案
参考窗体_销售员图.cs的实现，在窗体_利润统计.cs中添加部门筛选业务员的功能。

## 实现步骤

### 1. 修改窗体_利润统计.cs

#### 1.1 修改窗体加载事件 (窗体_利润统计_Load)
- 将业务员列表的数据源绑定改为使用 DefaultView
- 这样可以通过设置 RowFilter 来动态筛选

**修改前:**
```csharp
列表_业务员.DataSource = 模块_通用函数.易捷业务员表();
列表_业务员.DisplayMember = "EMPNME";
列表_业务员.ValueMember = "EMPCDE";
```

**修改后:**
```csharp
列表_业务员.DataSource = 模块_通用函数.易捷业务员表().DefaultView;
列表_业务员.DisplayMember = "EMPNME";
列表_业务员.ValueMember = "EMPCDE";
```

#### 1.2 添加部门列表的 ItemCheck 事件处理
添加 列表_部门_ItemCheck 方法，实现当部门勾选状态变化时筛选业务员列表：

```csharp
private void 列表_部门_ItemCheck(object sender, ItemCheckEventArgs e)
{
    var count = 列表_部门.CheckedItems.Count;
    var 当前项 = 列表_部门.SelectedIndex;
    var 当前状态 = 列表_部门.GetItemChecked(当前项);
    if (当前状态) {
        count -= 1;
    } else {
        count += 1;
    }
    if (count > 0) {
        var tmpstr = "";
        for (int i = 0; i < 列表_部门.Items.Count; i++) {
            var 该项状态 = 列表_部门.GetItemChecked(i);
            if (i == 当前项) 该项状态 = !该项状态;
            if (该项状态 == true) {
                if (tmpstr != "") {
                    tmpstr += ",";
                }
                tmpstr += "'" + 列表_部门.GetItemText(列表_部门.Items[i]) + "'";
            }
        }
        模块_通用函数.易捷业务员表().DefaultView.RowFilter = "TEMNME in (" + tmpstr + ")";
    } else {
        模块_通用函数.易捷业务员表().DefaultView.RowFilter = "";
    }
}
```

### 2. 修改窗体_利润统计.Designer.cs

#### 2.1 为部门列表添加 ItemCheck 事件绑定
在 InitializeComponent 方法中找到 列表_部门 的初始化代码，添加事件绑定：

**添加:**
```csharp
this.列表_部门.ItemCheck += new System.Windows.Forms.ItemCheckEventHandler(this.列表_部门_ItemCheck);
```

## 技术要点

1. **使用 DefaultView**: 业务员表使用 DataTable.DefaultView 作为数据源，可以通过 RowFilter 属性动态筛选数据
2. **ItemCheck 事件**: 在 ItemCheck 事件中处理勾选状态变化，注意此时 CheckedItems 还未更新，需要手动计算新的勾选状态
3. **TEMNME 字段**: 部门名称字段，用于匹配业务员所属部门

## 预期效果
- 当用户勾选某个部门时，业务员列表只显示该部门的业务员
- 当用户取消所有部门勾选时，业务员列表显示所有业务员
- 多选部门时，业务员列表显示所有选中部门的业务员
