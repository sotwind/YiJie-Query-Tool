# 数据汇总为0问题分析计划

## 问题描述
销售员图表窗体中，点击查询后数据汇总行显示全为0，但列表中有业务员数据。

## 问题分析

### 1. 代码现状分析

当前`窗体_销售员图.cs`的查询逻辑：

```csharp
// 连接的是易捷集团数据库 (fgrp)
private const string 易捷集团连接字符串 = "Data Source=(DESCRIPTION=(ADDRESS=(PROTOCOL=TCP)(HOST=36.138.130.91)(PORT=1521))(CONNECT_DATA=(SERVICE_NAME=dbms)));User Id=fgrp;Password=kuke.fgrp;";

// SQL查询grp_ord_data表
sql = "select objtyp, agntcde, nvl(sum(accamt),0) as 金额, nvl(sum(acreage * ordnum),0) as 面积, count(*) as 单数 from grp_ord_data where status='Y'";
```

### 2. 发现的问题

#### 问题1：grp_ord_data表可能不存在于易捷集团数据库
- `grp_ord_data`是一个汇总表，通常存在于各工厂的生产数据库中（新厂、老厂、温森）
- 但代码中连接的是**易捷集团数据库** (`fgrp`)，该数据库主要用于组织架构数据
- 易捷集团数据库中可能没有`grp_ord_data`表，或者该表为空

#### 问题2：正确的数据来源
根据`数据库连接信息.md`：
| 数据库 | 主要用途 |
|--------|----------|
| 新厂新系统 | 生产数据、订单、报价 |
| 老厂新系统 | 生产数据、订单、报价 |
| 温森新系统 | 生产数据、订单、报价 |
| 易捷集团 | 组织架构、部门、业务员 |

订单数据应该在各工厂的`ferp`数据库中，而不是易捷集团的`fgrp`数据库。

### 3. 修复方案

有两种修复方案：

#### 方案A：查询各工厂数据库（推荐）
遍历所有工厂数据库（新厂、老厂、温森），分别查询`grp_ord_data`表，然后汇总数据。

#### 方案B：使用视图或汇总表
如果集团数据库中有汇总视图，可以直接查询视图。

### 4. 实施步骤

1. **确认grp_ord_data表位置**
   - 检查易捷集团数据库是否有grp_ord_data表
   - 确认各工厂数据库中grp_ord_data表的结构和数据

2. **修改查询逻辑**
   - 如果是方案A：遍历各工厂数据库查询并汇总
   - 如果是方案B：修改连接字符串指向正确的数据库

3. **测试验证**
   - 验证查询结果不为0
   - 验证汇总计算正确
