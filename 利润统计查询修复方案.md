# 利润统计查询问题分析和修复方案

## 问题核心

### 1. 字段名不匹配问题

**当前 SQL（错误）**:
```sql
LEFT JOIN pb_dept_member e ON t.agntcde = e.mobile
```

**问题**: 
- `pb_dept_member` 表中没有 `mobile` 字段
- 新系统 `pb_dept_member` 表的关键字段是：`user_cde` (业务员编码), `dept_cde` (部门编码)
- `ord_ct` 表的 `agntcde` 字段存储的是业务员编码（如手机号或员工号）

### 2. 新旧系统数据结构差异

#### 旧系统（临海等）
- 业务员表：`pb_emps` 
- 字段：`EMPCDE`, `EMPNME`, `TEMCDE`, `TEMNME`
- 筛选条件：`objtyp = 'AG'` (业务员)

#### 新系统（集团、老厂、新厂、温森）
- 业务员表：`pb_dept_member`
- 字段：`USER_CDE` (或 `EMPCDE`), `EMPNME`, `DPTCDE`, `DPTNME`
- 关联字段：`user_cde` 或 `empcde`（需要验证实际字段名）

### 3. 订单数据分布

根据您的说明：
- **易捷集团数据库 (fgrp)**: 可能没有详细订单数据
- **老厂新系统数据库 (ferp)**: 有部分订单数据
- **各子公司服务器**: 都有各自的订单数据

这意味着需要：
1. 从易捷集团数据库获取部门和业务员映射关系
2. 在各子公司数据库中查询订单数据
3. 使用正确的字段名进行关联

### 步骤 3: 测试正确的 JOIN 条件

**方案 A: 如果 pb_dept_member 有 user_cde 字段**
```sql
SELECT 
    TO_CHAR(b.created, 'yyyy-MM-dd') as 日期，
    b.serial as 单号，
    c.clntnme as 客户，
    b.prdnme as 产品，
    e.user_nme as 业务员，
    d.dept_nme as 部门，
    nvl(b.quoprc, 0) * nvl(b.accnum, 0) as 报价总金额，
    nvl(b.accamt, 0) as 卖价总金额，
    nvl(b.accamt, 0) - nvl(b.quoprc, 0) * nvl(b.accnum, 0) as 利润差额
FROM ord_bas b
LEFT JOIN pb_clnt c ON b.clntcde = c.clntcde
LEFT JOIN ord_ct t ON b.serial = t.serial
LEFT JOIN pb_dept_member e ON t.agntcde = e.user_cde
LEFT JOIN pb_dept d ON e.dept_cde = d.dept_cde
WHERE b.isactive = 'Y'
  AND b.created >= DATE '2026-03-04'
  AND b.created < DATE '2026-03-05'
  AND ROWNUM <= 20;
```

## 修复方案

### 修复 1: 窗体_利润统计.cs

```csharp
// 原 SQL（错误）
LEFT JOIN pb_dept_member e ON t.agntcde = e.mobile
LEFT JOIN pb_dept d ON e.dept_cde = d.dept_cde

// 修复后（需要验证字段名）
LEFT JOIN pb_dept_member e ON t.agntcde = e.user_cde
LEFT JOIN pb_dept d ON e.dept_cde = d.dept_cde

// 或者如果字段名是 empcde
LEFT JOIN pb_dept_member e ON t.agntcde = e.empcde
LEFT JOIN pb_dept d ON e.dept_cde = d.dept_cde
```

### 修复 2: 部门筛选条件

当前代码问题：
```csharp
// 问题：使用 rowview["TEMCDE"] 但可能为空
string 部门编码 = rowview["TEMCDE"].ToString();
```

需要确保 `模块_通用函数.易捷部门表 ()` 返回的 `TEMCDE` 字段有正确值。

根据 `模块_通用函数.cs` 中的代码，从易捷集团数据库查询时：
```sql
SELECT m.user_cde as EMPCDE, m.dept_cde as TEMCDE, m.dept_cde as TEMCDE2, 
       m.user_nme as EMPNME, d.dept_nme as TEMNME
FROM pb_dept_member m
LEFT JOIN pb_dept d ON m.dept_cde = d.dept_cde
WHERE m.isactive = 'Y'
ORDER BY d.dept_nme, m.user_nme
```

这个查询应该能正确获取 `TEMCDE`（部门编码）。

### 修复 3: 业务员筛选条件

当前代码（窗体_利润统计.cs）:
```csharp
// 错误：使用了 h.mobile，但 h 表 (hr_base) 没有被正确 JOIN
sql += @" AND h.mobile IN (" + tmpstr + @")";
```

应该改为：
```csharp
sql += @" AND t.agntcde IN (" + tmpstr + @")";
```

```sql
LEFT JOIN ferp.pb_dept_member e ON t.agntcde = e.user_cde
LEFT JOIN ferp.pb_dept d ON e.dept_cde = d.dept_cde

  AND b.created >= DATE '2026-03-04'
  AND b.created < DATE '2026-03-05'
ORDER BY b.created;
```
