## 六、建议修复

### 1. 销售员图 - 面积计算修复

```sql
-- 原 SQL
nvl(sum(t.acreage * t.ordnum),0) as 面积

-- 建议修改
nvl(sum(t.acreage * nvl(t.ordnum, 0)), 0) as 面积
```

### 2. 利润统计 - 利率计算优化

```sql
-- 当前 SQL 已有保护，但建议增加显示说明
case 
    when nvl(b.quoprc, 0) * nvl(b.accnum, 0) = 0 then null  -- 改为 NULL 而不是 0
    else (nvl(b.accamt, 0) - nvl(b.quoprc, 0) * nvl(b.accnum, 0)) / (nvl(b.quoprc, 0) * nvl(b.accnum, 0)) * 100
end as 利率
```

### 3. 部门筛选逻辑

当前代码中 `窗体_利润统计.cs` 使用 `pb_dept_member` 表关联：
```sql
LEFT JOIN pb_dept_member e ON t.agntcde = e.user_cde
LEFT JOIN pb_dept d ON e.dept_cde = d.dept_cde
```

**验证结果**: ✅ 表结构支持此关联，`PB_DEPT_MEMBER` 表包含 `USER_CDE` 和 `DEPT_CDE` 字段
